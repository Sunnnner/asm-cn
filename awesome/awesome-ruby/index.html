    Ruby

    真棒Ruby

    由社区驱动的分类，包括令人敬畏的Ruby库，工具，框架和软件. 构建现代应用程序和Web应用程序的基本Ruby.

    欢迎分享，建议和贡献！ 请看看 contribution guidelines and quality standard 第一.

    谢谢大家 contributors，你很棒，如果没有你，这是不可能的！

    Abstraction¶

    • ActiveInteraction - 管理特定于应用的业务逻辑.
    • Apotomo - 基于Cell，Apotomo为您提供小部件和封装，冒泡事件，AJAX页面更新，坚如磐石的测试等等.
    • Cells - 查看Rails的组件.
    • Decent Exposure - 用于在控制器中创建声明性接口的帮助程序.
    • Docile - 一个小型库，可让您快速将DSL（特定于域的语言）映射到Ruby对象.
    • dry-rb - dry-rb是下一代Ruby库的集合，每个库都旨在封装一个常见的任务.
    • Interactor - Interactor提供了一个通用接口，用于在单个请求中执行复杂的交互.
    • Light Service - 系列行动，重点是简单.
    • Mutations - 将业务逻辑编写为清理和验证输入的命令.
    • Rails Event Store (RES) - 用于发布，使用，存储和检索事件的库. 对于使用Rails应用程序的事件驱动架构，它是您最好的伴侣.
    • Responders - 一组Rails响应程序来干扰您的应用程序.
    • Surrounded - 在单个系统中封装相关对象以在运行时添加行为. 可扩展的DCI实施.
    • Trailblazer - 开拓者是一个在Rails上面的薄层. 它轻轻地实施封装，直观的代码结构，并为您提供面向对象的体系结构.
    • Waterfall - 一系列功能性编程，用于链接ruby服务和块，从而提供流量控制的新方法.
    • wisper - 一个为Ruby对象提供发布 - 订阅功能的微型库.

    Admin Interface¶

    • ActiveAdmin - Ruby on Rails框架，用于为网站管理创建优雅的后端.
    • ActiveScaffold - ActiveScaffold为Rails应用程序的CRUD（创建，读取，更新，删除）操作提供快速而强大的用户界面. 它非常适合生成管理界面，管理数据重型应用程序，创建小部件或快速原型设计. ActiveScaffold可完全自定义，并提供一系列附加功能，包括搜索，分页，布局控制以及字段，表单和模板的覆盖.
    • Administrate - 一个Rails引擎，可帮助您将Thoughtbot的超灵活管理仪表板组合在一起.
    • bhf - 一个简单易用的Rails-Engine-Gem，为受信任的用户提供管理界面.
    • RailsAdmin - Rails引擎，提供易于使用的界面来管理数据.
    • Trestle - 适用于Rails的现代响应式管理框架. 在几分钟内构建一个后端，随着应用程序的需求而增长.

    Analytics¶

    • Ahoy - 跟踪Ruby，JavaScript和本机应用程序中的访问和事件的坚实基础.
    • Analytical - 用于管理rails应用中的多种分析服务的Gem.
    • Gabba - 向Google Analytics发送服务器端通知的简单方法.
    • Impressionist - 跟踪展示次数和网页浏览量的Rails插件.
    • Legato - 针对官方Google Analytics Reporting API的模型分析报告和查询.
    • Rack::Tracker - 机架中间件，可以连接到多个服务并以统一的方式公开它们.
    • Staccato - 将分析跟踪到官方Google Analytics Collection API.

    API Builder and Discovery¶

    • ActiveModel::Serializers - 对象的JSON序列化.
    • Blanket - 一个死的简单API包装器.
    • Crepe - 瘦API堆栈.
    • Fast JSON API - 用于Ruby对象的快速JSON：API序列化程序.
    • Grape - 用于在Ruby中创建类似REST的API的自以为是的微框架.
    • Her - 将REST资源映射到Ruby对象的ORM. 旨在构建由RESTful API而非数据库提供支持的应用程序.
    • jbuilder - 通过Builder风格的DSL创建JSON结构.
    • jsonapi-rb - 高效便捷的JSON API（de）序列化库.
    • JSONAPI::Resources - JSONAPI :: Resources或“JR”提供了一个框架，用于开发符合JSON API规范的服务器.
    • JSONAPI::Utils - JSONAPI :: Utils构建于JSONAPI :: Resources之上，利用其资源驱动的风格，并提供了一种简单的方法来构建现代JSON API，没有或没有更少的学习曲线.
    • Jsonite - 适用于API的小型，符合HAL标准的JSON演示者.
    • Pliny - 用于在Ruby中编写优秀API的Opinionated模板Sinatra应用程序.
    • rabl - 使用json，bson，xml，plist和msgpack支持的一般ruby模板.
    • Rails::API - 仅限API应用程序的Rails.
    • Roar - Ruby中面向资源的体系结构.
    • Spyke - 以类似ActiveRecord的方式与REST服务交互.
    • Version Cake - 在Rails应用程序中版本API的一种不显眼的方式.
    • versionist - 用于对基于Rails的RESTful API进行版本控制的插件.

    Assets¶

    • Asset Sync - 在Rails和S3之间同步资产.
    • Autoprefixer - 解析CSS并通过Can I Use将规则添加到规则中.
    • Bourbon - 轻量级Sass工具套装.
    • bower-rails - Bower对Rails项目的支持.
    • Emoji - 公开Phantom Open Emoji库unicode / image资产和API以便与它们一起使用.
    • Less Rails - Rails资产管道的动态样式表语言.
    • Less - Leaner CSS，在您的浏览器或Ruby中.
    • Rails Assets - Bundler到Bower代理.
    • Sass - Sass让CSS再次变得有趣.
    • Sprockets - 基于机架的资产包装系统.
    • Torba - 用于链轮的无鲍尔捆绑器.
    • Webpacker - 使用Webpack在Rails中管理类似app的JavaScript模块.

    Authentication and OAuth¶

    • Authlogic - Authlogic是一种干净，简单且不显眼的ruby身份验证解决方案.
    • Clearance - 针对Rails的小而简单的基于电子邮件和密码的身份验证.
    • Devise - 基于Warden的Rails灵活的身份验证解决方案.
    • JWT - Ruby中的JSON Web Token实现.
    • Knock - Rails API的无缝JWT身份验证.
    • Monban - 用于rails的非常简单且可扩展的用户身份验证库.
    • OmniAuth - 利用Rack中间件标准化多提供商身份验证的库.
    • Rodauth - Rack应用程序的身份验证和帐户管理框架.
    • Shield - 用于路由和模型上下文的身份验证协议.
    • Sorcery - Rails 4和Rails的神奇身份验证5.
    • warden - 通用机架认证框架.
    • OAuth：
    • Doorkeeper - Rails的OAuth2提供程序.
    • OAuth2 - OAuth 2.0协议的Ruby包装器.

    Authorization¶

    • acl9 - Acl9是一个基于角色的授权系统，它提供了一个简洁的DSL来保护您的Rails应用程序.
    • AccessGranted - Rails的基于多角色和白名单的授权gem.
    • Authority - ORM中立的方式来授权您的Rails应用程序中的操作.
    • CanCanCan - 继续使用CanCan，Ruby on Rails的授权Gem.
    • Declarative Authorization - 授权Rails插件，使用声明性DSL在一个地方指定授权规则.
    • Pundit - 通过OO设计和纯Ruby类的最小授权.

    Automation¶

    • Danger - 自动化团队围绕代码审查的惯例.
    • Huginn - Huginn是一个用于构建代理的系统，可以在线为您执行自动化任务.

    Caching¶

    • Action caching for Action Pack - Action Pack的动作缓存.
    • Dalli - 用于访问memcached服务器的高性能纯Ruby客户端.
    • Garner - 一组Rack中间件和缓存助手，实现各种缓存策略.
    • IdentityCache - 插入ActiveRecord的blob级缓存解决方案.
    • Kashmir - Kashmir是一个Ruby DSL，可以快速地序列化和缓存对象.
    • Readthis - 使用Redis实现高性能，池化，ActiveSupport兼容的缓存.
    • Record Cache - 在Rails中缓存活动模型记录3.
    • redis-store - Redis Store为现代Ruby框架提供了一整套商店（Cache，I18n，Session，HTTP Cache），例如：Ruby on Rails，Sinatra，Rack，Rack :: Cache和I18n. 它支持对象编组，超时，单个或多个节点以及命名空间.
    • Second Level Cache - ActiveRecord 4的Write-Through和Read-Through缓存库.

    Captchas and anti-spam¶

    • Invisible Captcha - 基于蜜罐策略的不引人注目且灵活的垃圾邮件防护. 它还提供时间敏感的表单提交.
