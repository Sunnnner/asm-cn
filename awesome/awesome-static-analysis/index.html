    静态分析和代码质量

    >静态程序分析是在没有实际执行程序的情况下执行的计算机软件分析 - Wikipedia

    这是静态分析工具和代码质量检查器的集合. 非常欢迎拉动请求！

    ：copyright：代表专有软件. 所有其他工具都是开源的. ：警告：表示社区不建议使用此工具 新项目已经过时或不再维护.

    Programming Languages¶

    Abap¶

    • abapOpenChecks - 使用新的和可自定义的检查增强SAP Code Inspector.

    Ada¶

    • Codepeer - 检测运行时和逻辑错误
    • Polyspace for Ada ：copyright： - 提供代码验证，证明没有溢出，被零除，越界数组访问以及源代码中的某些其他运行时错误.
    • SPARK ：copyright： - Ada的静态分析和形式验证工具集
    • Understand ：copyright： - 为Ada和VHDL提供代码分析，标准测试，度量，图形，依赖性分析等的IDE.

    Awk¶

    • gawk --lint - 警告其他awk实现可疑或不可移植的构造.

    C/C++¶

    • CBMC - bounded model-checker for C programs, user-defined assertions, standard assertions, several coverage metric analyses
    • clang-tidy - 铿锵静态分析
    • CMetrics - 测量C文件的大小和复杂性
    • Codecheker - 使用Web GUI对C / C ++代码进行静态分析
    • CodeSonar from GrammaTech ：copyright： - 高级，完整的程序，深入的​​路径，C和C ++的静态分析，具有易于理解的解释和代码以及路径可视化.
    • Corrode - 从C到Rust的半自动翻译. 可以通过显示Rust编译器警告和错误来揭示原始实现中的错误.
    • cppcheck - C / C ++代码的静态分析
    • CppDepend ：copyright： - 测量，查询和可视化您的代码，避免意外问题，技术债务和复杂性.
    • cpplint - 遵循Google风格指南的自动C ++检查程序
    • cqmetrics - C代码的质量指标
    • CScout - C和C预处理器代码的复杂性和质量指标
    • flawfinder - 发现可能存在的安全漏洞
    • flint++ - flint的跨平台，零依赖端口，在Facebook开发和使用的C ++的lint程序.
    • Frama-C - 用于C代码的声音和可扩展静态分析器
    • Helix QAC ：copyright： - 嵌入式软件的企业级静态分析. 支持MISRA，CERT和AUTOSAR编码标准.
    • IKOS - 基于LLVM的C / C ++代码的声音静态分析器
    • include-gardener - a static analyzer for C/C++/Obj-C to create a graph (in dot or graphml format) which shows all #include relations of a given set of files.
    • oclint - C / C ++代码的静态分析
    • Polyspace Bug Finder ：copyright： - 识别C和C ++嵌入式软件中的运行时错误，并发问题，安全漏洞和其他缺陷.
    • Polyspace Code Prover ：copyright： - 提供代码验证，证明在C和C ++源代码中没有溢出，被零除，越界数组访问以及某些其他运行时错误.
    • scan-build - 在编译时使用LLVM分析C / C ++代码
    • splint - 注释辅助静态程序检查器
    • vera++ - Vera ++是一个用于验证，分析和转换C ++源代码的可编程工具.

    C#¶

    • .NET Analyzers - 使用.NET编译器平台开发分析器（诊断和代码修复）的组织.
    • Code Analysis Rule Collection - 包含一组基于Microsoft .NET编译器平台“Roslyn”构建的诊断，代码修复和重构.
    • code-cracker - An analyzer library for C# and VB that uses Roslyn to produce refactorings, code analysis, and other niceties.
    • CodeRush ：copyright： - 在Visual Studio 2015及更高版本中使用Roslyn引擎的代码创建，调试，导航，重构，分析和可视化工具.
    • CSharpEssentials - C# Essentials is a collection of Roslyn diagnostic analyzers, code fixes and refactorings that make it easy to work with C# 6 language features.
    • Designite ：copyright： - Designite是一种软件设计质量评估工具. 它支持检测实现和设计气味，计算各种代码质量指标和趋势分析.
    • Gendarme - Gendarme检查包含ECMA CIL格式（Mono和.NET）代码的程序和库.
    • NDepend ：copyright： - 测量，查询和可视化您的代码，避免意外问题，技术债务和复杂性.
    • Puma Scan - 随着开发团队在Visual Studio中编写代码，Puma Scan为常见漏洞（XSS，SQLi，CSRF，LDAPi，加密，反序列化等）提供实时安全代码分析.
    • Refactoring Essentials - The free Visual Studio 2015 extension for C# and VB.NET refactorings, including code best practice analyzers.
    • ReSharper - Extends Visual Studio with on-the-fly code inspections for C#, VB.NET, ASP.NET, JavaScript, TypeScript and other technologies.
    • Roslyn Security Guard - 专注于识别潜在漏洞的项目，例如SQL注入，跨站点脚本（XSS），CSRF，加密弱点，硬编码密码等等.
    • Roslynator - A collection of 190+ analyzers and 190+ refactorings for C#, powered by Roslyn.
    • Security Code Scan - Security code analyzer for C# and VB.NET. Detects various security vulnerability patterns: SQLi, XSS, CSRF, XXE, Open Redirect, etc.
    • SonarLint for Visual Studio - SonarLint是Visual Studio 2015和2017的扩展，它为开发人员提供了有关注入.NET代码的新错误和质量问题的即时反馈.
    • VSDiagnostics - 基于Roslyn的静态分析器集合，与VS集成.
    • Wintellect.Analyzers - .NET编译器平台（“Roslyn”）诊断分析器和代码修复程序.

    • crystal - Crystal编译器具有内置的linting功能.

    Dlang¶

    Elixir¶

    • credo - 静态代码分析工具，侧重于代码一致性和教学.
    • sobelow - Phoenix Framework的以安全为中心的静态分析

    Erlang¶

    • elvis - Erlang Style Reviewer

    F#¶

    • FSharpLint - F的棉绒工具#

    Fortran¶

    • i-Code CNES for Fortran - Fortran 77，Fortran 90和Shell的开源静态代码分析工具.

    Go¶

    • deadcode - 查找未使用的代码.
    • dingo-hunter - 用于在Go中查找死锁的静态分析器.
    • dupl - 报告可能重复的代码.
    • errcheck - 检查是否使用了错误返回值.
    • flen - 获取Go包中函数长度的信息.
    • gas - 通过扫描Go AST检查安全问题的源代码.
    • Go Meta Linter - 同时运行Go lint工具并规范化输出.
    • go tool vet --shadow - 报告可能无意中被遮蔽的变量.
    • go vet - 检查Go源代码并报告可疑.
    • go-consistent - 分析器，可帮助您使Go程序更加一致.
    • go-critic - 转到源代码linter，维护当前未在其他链接中实现的检查.
    • go-staticcheck - go vet on steroids, similar to ReSharper for C#.
    • go/ast - 包ast声明用于表示Go包的语法树的类型.
    • goconst - 查找可以用常量替换的重复字符串.
    • gocyclo - 计算Go源代码中函数的圈复杂度.
    • gofmt -s - 检查代码是否格式正确，无法进一步简化.
    • goimports - 检查丢失或未引用的包导入.
    • GolangCI-Lint - “Go Meta Linter”的替代方案：GolangCI-Lint是一个直接聚合器.
    • golint - 在Go源代码中打印出编码风格错误.
    • goreporter - 同时运行许多短路并将其输出规范化为报告.
    • goroutine-inspect - 分析Golang goroutine转储的交互式工具.
    • gosimple - 报告代码简化.
    • gotype - 类似于Go编译器的语法和语义分析.
    • ineffassign - 检测Go代码中的无效分配
    • interfacer - 建议可以使用的更窄的接口.
    • lll - 报告长线.
    • maligned - 检测在字段排序时占用较少内存的结构.
    • megacheck - 运行staticcheck，gosimple和unused，共享工作.
    • misspell - 查找通常拼写错误的英语单词.
    • nakedret - 找到裸体回报.
    • prealloc - 查找可能预先分配的切片声明.
    • revive - Go的快速，可配置，可扩展，灵活和美观的linter. 直接替换golint.
    • safesql - Golang的静态分析工具，可防止SQL注入.
    • structcheck - 查找未使用的结构字段.
    • test - 从stdlib测试模块显示测试失败的位置.
    • testify - 显示失败的证词断言的位置.
    • unconvert - Detect redundant type conversions.
    • unimport - 查找不必要的导入别名
    • unparam - 查找未使用的功能参数.
    • unused - 查找未使用的变量.
    • varcheck - 查找未使用的全局变量和常量.

    Groovy¶

    • CodeNarc - Groovy源代码的静态分析工具，支持监控和实施许多编码标准和最佳实践

    Haskell¶

    • HLint - HLint是一个用于建议对Haskell代码进行可能改进的工具.
    • Weeder - 用于在Haskell代码中检测死导出或包导入的工具.

    Haxe¶

    • Haxe Checkstyle - 一个静态分析工具，可帮助开发人员编写符合编码标准的Haxe代码.

    Java¶

    • ArchUnit - 对Java体系结构进行单元测试
    • Checker Framework - Java的可插入类型检查http://checkerframework.org/
