    Tools¶

    Analyzers¶

    • Confess - 使用PhantomJS无头分析网页并生成清单.
    • DebugBear - DebugBear是一个基于Lighthouse的网站监控工具. 了解您的分数和指标如何随时间变化，重点是了解导致每项变化的因素. DebugBear是一款付费产品，可免费试用30天.
    • Page Speed - PageSpeed系列工具旨在帮助您优化网站的性能. PageSpeed Insights产品将帮助您确定可应用于您网站的性能最佳实践，而PageSpeed优化工具可帮助您自动化流程.
    • Lighthouse - 渐进式Web应用程序的审计和性能指标工具.
    • YSlow - YSlow分析网页，并根据一组高性能网页规则提出改进其性能的方法.
    • YSlow for PhantomJS - 用于PhantomJS的YSlow还为自动化测试框架引入了新的输出格式：TAP（测试任何协议）和JUnit.
    • Grunt-WebPageTest - 用于连续测量WebPageTest的Grunt插件. （Demo)
    • Grunt-yslow - 使用PhantomJS（无头WebKit浏览器）测试页面性能的Grunt任务.
    • Grunt-perfbudget - 用于执行绩效预算的Grunt.js任务（more on performance budgets).
    • Web Tracing Framework - Web跟踪框架是一组用于跟踪和调查复杂Web应用程序的库，工具和可视化工具
    • HP LoadRunner - Hewlett-Packard的自动化性能和测试自动化产品，用于应用程序负载测试：检查系统行为和性能，同时生成实际负载.
    • Yandex.Tank - 面向高级Linux用户的可扩展开源负载测试工具，作为自动负载测试套件的一部分特别好.
    • Yellow Lab Tools - 在线快速简便的工具，可以审核前端不良做法，揭示性能问题和配置文件JavaScript.
    • Pagelocity - Web性能优化和分析工具.
    • Varvy - 测试您的网站，看它是否符合Google的速度指南.
    • Web Bloat Score Calculator - 将页面大小与同一页面的压缩图像进行比较
    • Speed Racer - 使用Chrome headless收集库/应用程序的性能指标.
    • Speedrank - Speedrank在后台监控您网站的性能. 它会随着时间的推移显示Lighthouse报告并提供改进建议. Speedrank是一款带有14天试用版的付费产品.

    Analyzers - API¶

    • Node-yslowjs - Node.js上的YSlow.js是一个简单的Node.js包装器，用于以编程方式运行phantomjs yslow.js.
    • PSI - 针对节点的PageSpeed见解 - 包含报告.

    Bundle Analyzer¶

    • Bundlesize - 检查您的捆绑尺寸.
    • source-map-explorer - 通过源映射分析和调试捆绑空间使用情况.
    • Bundlephobia - 查找将npm包添加到捆绑包的成本.
    • Disc - 可视化browserify项目包的模块树并跟踪膨胀.
    • lasso-analyzer - 分析和可视化由Lasso创建的项目包.

    Benchmark - CSS¶

    • CSS-perf - 完全不科学的测试CSS性能的方法. 大多数这些测试将围绕确定有效CSS架构的方法和技术展开. 换句话说，我想知道在CSS策略的特定比较中最有效的方法.

    Benchmark - Javascript¶

    >一组用于创建测试用例并比较JavaScript中的不同实现的工具.

    • JSPerf - jsPerf旨在提供一种简单的方法来创建和共享测试用例，通过运行基准来比较不同JavaScript代码段的性能.
    • Benchmark.js - 强大的基准测试库，适用于几乎所有JavaScript平台，支持高分辨率计时器，并返回统计上显着的结果.
    • JSlitmus - JSLitmus是一个用于创建临时JavaScript基准测试的轻量级工具.
    • Matcha - Matcha允许您设计测量代码性能的实验. 建议每个工作台都关注您应用中的特定影响点.
    • Timing.js - Timing.js是一小组帮助程序，用于使用Navigation Timing API来识别应用程序花费时间的位置. 可用作独立脚本，DevTools Snippet或bookmarklet.
    • Stats.js - 此类提供了一个简单的信息框，可帮助您监控代码性能.
    • PerfTests - JavaScript继承模型的性能测试.
    • Memory-stats.js - 通过性能内存最小化JS堆大小监视器.

    Benchmark - PHP¶

    • PHPench - PHPench为PHP基准测试创建图形输出. 使用GnuPlot实时绘制任何函数的运行时，并从结果中创建一个图像.
    • php-bench - 测量PHP代码块的基准和配置文件，同时测量性能足迹.

    Bookmarklets¶

    • Yahoo YSlow for Mobile/Bookmarklet - YSlow分析网页，并根据一组高性能网页规则提出改进其性能的方法.
    • PageSpeed [已弃用] - PageSpeed Insights可作为Google Chrome的开源浏览器扩展程序提供. 网站管理员和网站开发人员可以使用PageSpeed Insights来评估其网页的效果，并获得有关如何改进网页的建议 - 解决方法 @Kiquenet - https://gist.github.com/davidsonfellipe/f8e3723b4dc0aaf9f6d4.
    • PerfMap - 使用Resource Timing API创建浏览器中加载的资源的前端性能热图的书签.
    • DOM Monster - 一个跨平台的跨浏览器书签，它将分析您所在页面的DOM和其他功能，并为您提供健康状况.
    • CSS Stress - CSS Stress是一种测试和性能分析.
    • Performance-Bookmarklet - 通过Resource Timing API，Navigation Timing API和User-Timing分析当前页面 - 轻量级实时WebPageTest的排序. 如 Chrome Extension 和 Firefox Add-on 名为Performance-Analyzer.

    CDN¶

    >内容传送网络或内容分发网络（CDN）是部署在因特网上的多个数据中心中的大型分布式服务器系统. CDN的目标是为具有高可用性和高性能的最终用户提供内容. 查看大量的CDN列表 Wikipedia.

    • jsDelivr - 与Google Hosted Libraries类似，jsDelivr是一个开源CDN，允许开发人员托管他们自己的项目，任何人都可以链接到他们网站上的托管文件.
    • CDNjs - 由CloudFlare赞助的Javascript和CSS的开源CDN，托管从jQuery和Modernizr到Bootstrap的所有内容.
    • jQuery - jQuery CDN - 最新的稳定版本，由MaxCDN提供支持. ：cn： UpYun CDN - 由upyun提供的CDN. ：cn： Sina Public Resources - CDN由sinaapp.com提供. *：cn： Bootstrap中文网开放CDN服务 - Bootstrap中文网络开放CDN服务（仅限HTTP）.

    >要找到有用的更多信息，请在付费CDN之间做出正确的选择，请访问 CDNPlanet.

    CDN - Monitor¶

    • CDNperf - 找到快速可靠的JavaScript CDN，让您的网站快速而愉快.

    CDN - Utilities¶

    • Cdnex - 使用自定义CDN网址添加资产网址.
    • Gulp-google-cdn - 用Google CDN替换脚本引用.

    Extensions¶

    • Browser Calories - 衡量绩效预算的最简单方法.

    Image Optimizers¶

    >如何删除所有这些不必要的数据，并在不降低质量的情况下为您提供文件.

    • Grunt-smushit - 使用Yahoo Smushit删除不必要的PNG和JPG字节的Grunt插件.
    • Gulp-smushit - 使用Yahoo Smushit优化PNG和JPG的Gulp插件. 在烟雾之上制作.
    • Smush it - Smush它使用特定于图像格式的优化技术来从图像文件中删除不必要的字节. 它是一种“无损”工具，这意味着它可以在不改变外观或视觉质量的情况下优化图像.
    • Imagemin - 使用Node.js无缝缩小图像.
    • Sharp - 这个高速Node.js模块的典型用例是将多种格式的大图像转换为不同尺寸的小型，Web友好的JPEG，PNG和WebP图像.
    • Gm - GraphicsMagick和ImageMagick for node.
    • Exexif - 纯粹的elixir库从jpeg文件中提取tiff和exif元数据.
    • OptiPNG - OptiPNG是一种PNG优化器，可将图像文件重新压缩为更小的尺寸，而不会丢失任何信息.
    • Grunt-contrib-imagemin - 为Grunt缩小PNG和JPEG图像.

    Specs¶

    • Web Performance Working Group - 作为Rich Web Client Activity的一部分，Web性能工作组的任务是提供测量用户代理功能和API的应用程序性能方面的方法.
    • Page Visibility - 此规范定义了一种方法，供站点开发人员以编程方式确定页面的当前可见性状态，以便开发功能强大且CPU效率高的Web应用程序.
    • Navigation Timing - 该规范定义了用于存储和检索与文档导航相关的高分辨率性能度量数据的统一接口.
    • Resource Timing - 该规范定义了Web应用程序的接口，以访问文档中资源的完整定时信息.
    • User Timing - 此规范定义了一个界面，可帮助Web开发人员通过访问高精度时间戳来衡量其应用程序的性能.
    • Performance Timeline - 该规范定义了用于存储和检索性能度量数据的统一接口. 本规范未涵盖各个性能指标接口.
    • CSS will-change - 此规范定义了will-change CSS属性，该属性允许作者提前声明将来可能更改的属性，因此UA可以在需要之前的某个时间设置适当的优化. 这样，当实际发生更改时，页面会以一种快速的方式更新.
    • Resource Hints - 此规范定义HTML链接元素（<link>）的dns-prefetch，preconnect，prefetch和prerender关系. 这些原语使开发人员以及生成或传递资源的服务器能够帮助用户代理处理它应该连接到哪个源的决策过程，以及它应该获取和预处理哪些资源以提高页面性能.
