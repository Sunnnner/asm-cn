    >一系列令人敬畏的ESLint配置，插件等.

    如果您想贡献，请阅读 contribution guidelines.

    Configs¶

    • Airbnb - 可共享配置 Airbnb's style guide.
    • Canonical - 可共享配置 Canonical style guide.
    • ESLint - 包含用于ESLint团队维护的项目的ESLint配置.
    • ES - 可共享的配置，非常严格的代码.
    • Facebook - Facebook风格指南的可共享配置.
    • Google - 可共享配置 Google style.
    • Problems - 可共享配置，仅捕获实际问题，并不强制风格首选项.
    • Shopify - 可共享配置 Shopify's style guide.
    • Standard - JavaScript的可共享配置 Standard Style.
    • Supermind - Supermind风格的可共享配置.
    • XO - 可共享配置 XO.

    Parsers¶

    • Babel - 使用Babel的解析器来绘制所有Babel特征.
    • TypeScript - TypeScript解析器，生成与ESLint兼容的输出.
    • BrightScript - 用于Roku开发的BrightScript插件. 包括解析器和规则.

    Frameworks and Libraries¶

    • Angular - Linting规则坚持 John Papa's Angular Styleguide.
    • AVA - AVA的Linting规则.
    • Backbone - Backbone的Linting规则.
    • Chai - 柴的Linting规则.
    • Ember - Ember的Linting规则.
    • GraphQL - 根据模式检查GraphQL查询字符串.
    • Hapi - hapi的Linting规则.
    • Jasmine - 茉莉花的Linting规则.
    • Jest - Is的Linting规则.
    • JSDoc - JSDoc评论的Linting规则.
    • Lodash - Lodash特定的linting规则.
    • Lodash/fp - Lodash / fp特定的linting规则.
    • Lodash template - Lodash模板/下划线模板的插件.
    • Meteor - 流星特定的linting规则.
    • Mocha - Mocha的Linting规则.
    • Mongodb - Mongodb原生Node.js驱动程序linting规则.
    • Ramda - Ramda特定的linting规则.
    • React - React和JSX的Linting规则.
    • React Native - React Native特定的linting规则.
    • RequireJS - RequireJS的Linting规则.
    • VueJS - VueJS插件.

    Misc¶

    • Babel - 添加内置规则的替换以包括Babel功能.
    • Compat - 使用的API的Lint浏览器兼容性（caniuse 作为ESLint插件）.
    • Coffee - 使用Coffeelint库中的可选linting规则启用CoffeeScript文件的linting.
    • deprecate - 将函数或模块标记为已弃用，并在使用时获取lint消息.
    • disable - 使用文件路径模式和内联注释禁用指定的插件.
    • es5 - ES5用户的ESLint插件（禁止ES2015 +使用）.
    • Flow - 流动类型的linting规则.
    • Flow Errors - 将Flow作为ESLint插件运行.
    • HTML - 在HTML中使用Linting for JavaScript<script> tags.
    • import - 使用ES2015 +导入/导出语法，并防止拼写错误的文件路径和导入名称.
    • JSON - 提示您的JSON文件.
    • Markdown - Markdown中的Linting JavaScript.
    • Node - Node.js的Linting规则（检查导入路径，ES语法，......）.
    • Notice - 一个用于检查文件顶部并修复它们的eslint规则！
    • Optimize Regex - 优化正则表达式文字.
    • SonarJS - 检测错误和可疑模式的规则.
    • SQL - ESLint的SQL linting规则.
    • TypeLint - 介绍基于现有模式（Swagger，Redux）的类型和对对象属性的linting访问，防止出现“未定义”错误.
    • unicorn - 各种令人敬畏的ESLint规则.
    • ESLint Comments - 关于ESLint指令注释的最佳实践（/ * eslint-disable * /等等）.
    • eslint-plugin-eslint-plugin - 用于lint ESLint插件的ESLint插件.

    Practices¶

    • array-func - 使用es2015数组方法和函数时避免冗余.
    • fp - 函数式编程的ESLint规则.
    • Immutable - 禁用JavaScript中的所有突变.
    • JSX a11y - JSX元素的可访问性规则.
    • new-with-error - 要求使用new抛出错误.
    • no-inferred-method-name - ESLint的自定义规则，用于检查对象文字中的推断方法名称.
    • no-loops - 这是2017年，你仍然使用循环？
    • no-use-extend-native - 防止使用扩展的本机对象.
    • Promise - 使用承诺时的最佳做法.
    • Security - 节点安全的ESLint规则.
    • Simple import sort - 易于自动修复的导入排序.
    • this - 写纯函数，不要'this`.
    • XSS - 尝试在代码库结束生产之前检测XSS问题.

    Style¶

    • filenames - 确保JavaScript文件的文件名一致.

    Preconfigured Tools with ESLint Set up¶

    • Node.js Standard Style - Node.js核心配置.
    • prettier-standard - 允许使用自定义限制规则的更漂亮的格式化程序.
    • Standard - JavaScript标准风格.
    • Superlint - JavaScript Supermind风格.
    • XO - JavaScript幸福风格linter❤️.
    • Zoe - 通用JavaScript linter和formatter.
    • Healthier - 代码风格不可知的标准版，是Prettier的完美伴侣.

    Tools¶

    • eslint-cli - 这是执行本地安装的ESLint的eslint命令.
    • eslint-find-rules - 查找自定义配置中没有的内置ESLint规则.
    • eslint-index - 用于查找和管理ESLint配置文件中的规则的CLI.
    • eslint-multiplexer - Multiplex eslint结果和合并公共文件的结果.
    • eslint-nibble - 通过一次修复一条规则，轻松进入ESLint.
    • eslint-rule-documentation - 找到ESLint规则文档的URL.
    • eslint-watch - 使用监视模式运行ESLint.
    • codacy-eslint - Docker用于 Codacy 运行ESLint.
    • esprint - 跨多个线程运行ESLint.
    • eslint-formatter-gitlab - 输出ESLint导致GitLab代码质量结果.

    Developing for ESLint¶
